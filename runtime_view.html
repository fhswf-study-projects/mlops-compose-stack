This scenario describes how a user interacts with the system via the Streamlit UI to request a machine learning prediction. The User give also a feedback, if the prediction was right or not.

Prediction Interaction

  1. The user inputs data via the Streamlit UI.

  2. The Streamlit UI sends the request to the FastAPI backend.

  3. The FastAPI backend forwards the request to the Data Processor.

  4. The Data Processor loads the latest model from MLflow.

  5. The Data Processor preprocesses the data and performs inference.

  6. The prediction is returned to the FastAPI backend.

  7. The FastAPI backend sends the prediction result back to the Streamlit UI.

  8. The user sees the predicted result.

Feedback Interaction

  1. After receiving the prediction, the user provides feedback (correct or incorrect) through the Streamlit UI.

  2. The feedback is sent to FastAPI via the backend, which forwards it to Prometheus.

  3. Prometheus collects feedback metrics (e.g., accuracy) and monitors them.

  4. Grafana visualizes the feedback and tracks the metrics over time, monitoring whether the threshold is met (for example, if the accuracy drops below a certain threshold). Grafana also send a webhook to the API, if the threshold met a specific value.

  5. Threshold Detection: If the feedback threshold is reached (for example, if a certain number of incorrect predictions occur), Prometheus alerts the system, triggering model retraining.

  6. Retraining the Model: The Data Processor retrains the model and logs the new model version in MLflow.
